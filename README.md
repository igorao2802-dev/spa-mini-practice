## 🎤 Ответы на вопросы интервью

### 1. Что такое «однонаправленный поток данных» в React? Почему это удобно?

Данные в React передаются только сверху вниз через props. Это делает поток данных предсказуемым: легко отследить, откуда пришло значение и кто может его изменить. Упрощает отладку и тестирование.

### 2. Что такое lifting state up? Приведите пример из Вашей практической работы.

«Подъём состояния» — перенос state в общего родителя, когда данные нужны нескольким компонентам. Пример: `filter` и `bookings` хранятся в `App.js`, а `StatusFilter` и `BookingList` получают их через props и сообщают о событиях через callback.

### 3. Как дочерний компонент сообщает родителю о событии? Почему нельзя изменить state родителя напрямую?

Через callback-функцию, переданную через props. Прямое изменение state родителя нарушило бы однонаправленный поток и сделало бы данные непредсказуемыми. React требует явного запроса на изменение через функцию.

### 4. Что такое управляемый компонент (controlled component)? Зачем нужен атрибут value у input?

Управляемый компонент — input, значение которого хранится в state React. Атрибут `value` связывает поле с state, а `onChange` обновляет его. Это даёт полный контроль: валидация, очистка, синхронизация с другими полями.

### 5. Что такое паттерн «компонент-обёртка» (wrapper component)? Как работает children?

Обёртка задаёт общий каркас (шапка, структура), а `children` — специальный prop, куда вкладывается содержимое извне. Пример: `Layout.jsx` оборачивает всё приложение, обеспечивая единый дизайн без дублирования кода.

### 6. Почему в React предпочтительна композиция, а не наследование?

Композиция (вложение компонентов) гибче: можно комбинировать поведение без жёсткой иерархии. Наследование создаёт хрупкие связи, сложно тестировать и переиспользовать. React поощряет «собирать» UI из мелких блоков.

### 7. Зачем нужен useEffect для работы с localStorage? Что такое «побочный эффект» (side effect)?

`useEffect` выполняется после рендера — это правильное место для операций вне React-дерева (запросы, localStorage). «Побочный эффект» — любое взаимодействие с внешним миром, которое не влияет на возврат JSX.
